**Дата:** 15 декабря 2025 г.

**Статус:** Принято

**Контекст:**

Микросервисная система требует:
- Мониторинга здоровья микросервисов
- Трассировки запросов через несколько сервисов
- Централизованного логирования
- Оповещений при проблемах
- Дашбордов для операционной команды
- Хранения логов в соответствии с требованиями безопасности

**Рассмотренные варианты:**

1. **ELK Stack** – мощная система для логирования
2. **Prometheus + Grafana** – популярный стек для метрик
3. **Datadog** – полнофункциональный SaaS
4. **Jaeger + Prometheus + ELK** – комплексное решение
5. **Управляемые решения** – российские облачные сервисы

**Решение:**

Выбрана комбинация инструментов:
- **Prometheus + Grafana** для метрик и оповещений
- **ELK Stack** или **Loki + Grafana** для логирования
- **Jaeger** для распределённой трассировки
- **OpenTelemetry SDK** для инструментации приложений

**Обоснование:**

- **Prometheus + Grafana:** сбор метрик, дашборды, интеграция с Kubernetes
- **ELK Stack для логирования:** хранение, поиск, анализ логов
- **Jaeger для трассировки:** отслеживание запросов, визуализация
- **OpenTelemetry:** единый стандарт инструментации

**Недостатки решения:**

- **Операционная сложность:** несколько компонентов
- **Место для логов:** требуется политика хранения
- **Накладные расходы:** требует ресурсов
- **Кривая обучения:** время на освоение

**Последствия:**

- Инвестиции в DevOps для настройки
- Инструментация приложений OpenTelemetry
- Дашборды и оповещения по метрикам
- Документация для операционной команды
- Политика хранения логов и метрик
- Возможный переход на управляемое решение
