**Дата:** 15 декабря 2025 г.

**Статус:** Принято

**Контекст:**

Система должна хранить:
- Экспортируемые отчёты
- Графики потребления
- Кэшированные аналитические данные
- Логи и резервные копии
- Данные в российских дата-центрах
- Требуется долгосрочное и дешёвое хранилище

**Рассмотренные варианты:**

1. **Yandex Object Storage** – S3-совместимое облачное хранилище
2. **AWS S3** – популярное, но размещение вне РФ
3. **MinIO** – self-hosted решение
4. **VK Cloud Object Storage** – российский аналог S3
5. **Сетевое хранилище** – меньше гибкости

**Решение:**

Выбран **Yandex Object Storage** с многоуровневой политикой хранения.

**Обоснование:**

- **Yandex Object Storage:** российское решение, S3-совместимый API
- **Многоуровневая политика:** Standard, Cold, Ice для разных сценариев доступа
- **Интеграция с .NET:** через AWSSDK.S3
- **Масштабируемость:** автоматическое масштабирование

**Недостатки решения:**

- **Зависимость от провайдера:** риски недоступности
- **Задержки доступа к холодным данным:** время на восстановление
- **Дополнительные затраты:** мониторинг использования
- **Ограничения квот:** лимиты операций и размера объектов

**Последствия:**

- Настройка политик жизненного цикла
- Стратегия восстановления данных
- Безопасность: приватный доступ через временные URL
- Мониторинг использования и стоимости
- Настройка межрегиональной репликации при необходимости
