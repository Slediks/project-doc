**Дата:** 14 декабря 2025 г.

**Статус:** Принято

**Контекст:**

Система содержит несколько микросервисов, требующих:
- Высокой производительности
- Асинхронных операций
- Поддержки контейнеризации
- Взаимодействия с базами данных временных рядов
- Простоты развёртывания и мониторинга

**Рассмотренные варианты:**

1. **Java** – зрелый стек с большой экосистемой
2. **Kotlin** – современный язык для JVM
3. **Go** – быстрые бинарники, простая конкурентность
4. **C#** – кроссплатформенный стек с высокой производительностью
5. **Node.js + TypeScript** – быстрый цикл разработки

**Решение:**

Выбран **C#** для основных микросервисов с возможностью использования **Python** для аналитики.

**Обоснование:**

- **Высокая производительность:** ASP.NET Core
- **Встроенная асинхронность:** async/await
- **Dependency Injection:** упрощает разработку
- **Гибкость для микросервисов:** поддержка gRPC, HTTP/2
- **Контейнеризация:** отличная поддержка Docker
- **Интеграция с базами данных:** Entity Framework Core, Dapper
- **Наблюдаемость:** поддержка трассировки, логирования
- **Кроссплатформенность:** работает на Linux, Windows, macOS
- **Python для аналитики:** удобство интеграции ML библиотек

**Недостатки решения:**

- **Освоение стека:** требуется обучение
- **Меньшая популярность в стартапах**
- **Разнородный стек усложняет CI/CD**
- **Экосистема меньше, чем у Java**

**Последствия:**

- Инвестиции в инструменты DevOps
- Стандартизация паттернов логирования, трассировки
- CI/CD для разных типов проектов
- Обучение команды
- Унифицированный мониторинг
